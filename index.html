Zero to AI Infrastructure — the Forward Deployed Path

A complete, plain-English curriculum: GitHub & CI/CD → Linux, Docker & Kubernetes → cloud & Terraform → GPUs, LLM serving, MLOps, RAG, LoRA fine-tuning & AI agents — with a parallel track of data structures, algorithms and system design running through all 12 weeks. Twelve projects that climb in difficulty, every resource free (except ~$15 of GPU rental), and a built-in glossary that explains every term.

Assumes: some coding, zero infra
Pace: ~24 hrs/week (20 core + 4 parallel track)
Length: 12 weeks (~290 hrs)
Cost: ~$15 total (GPU hours)

The big picture — read this first

When you write code, someone still has to test it, package it, put it on computers on the internet, and keep it running when things break. The systems that do all of that are called infrastructure, and the people who build them are infrastructure engineers. When the thing being kept alive is an AI model — which runs on scarce, expensive chips called GPUs — the role is AI infrastructure engineering, and demand hugely outstrips supply.

This sprint is ordered so nothing ever assumes something you haven't done — never built an API, never written a test, never touched a database? Each is taught from zero in the week it first appears (weeks 2–3), with the exact tutorial linked. First you learn to control computers with text commands and manage code like a professional (week 1), then to build and ship software automatically (weeks 2–3), then to run it the way companies do (weeks 4–6), and only then — with all that scaffolding — do you touch GPUs, LLM serving, RAG, fine-tuning and agents (weeks 7–12). Skipping ahead is the #1 way beginners get confused: RAG tutorials feel like magic spells until you understand containers and APIs, and then they feel obvious.

Alongside the weekly phases runs a parallel track — data structures & algorithms (3 problems/week) and system design (one session/week) — because a great portfolio still won't survive a fumbled coding round or a blank whiteboard. It's designed to reinforce the sprint, not compete with it: the systems you design on paper are the ones you deployed the week before.

Two rules. Build more than you watch — after 2 hours of video, stop and type. And when any term reads like alphabet soup, use the Glossary at the bottom of this page — every buzzword on this page is explained there in plain English. Nothing on this page is assumed knowledge.

🎯 The end goal: Forward Deployed Engineer (FDE)

A Forward Deployed Engineer is the person an AI company sends to a customer to make the product actually work in the customer's messy, real-world environment. One week you're deploying a RAG system inside a bank's private cloud; the next you're fine-tuning a model on a client's data, wiring it into their Kubernetes cluster, and building the dashboard that proves it's worth the money. It's one of the fastest-growing, best-paid entry paths in AI right now — companies like Palantir, OpenAI and Anthropic hire aggressively for it.

What makes FDEs rare is breadth: they can touch every layer — infra, models, data, APIs, a bit of frontend — and they can explain it to non-engineers. That's exactly what this curriculum builds: not the deepest specialist in any one layer, but the engineer who can take "here's our data, make an AI product run in our environment" and deliver end-to-end. Every project in the ladder below is a rehearsal for that job.

🚀 After week 12 — the depth queue

Deliberately kept out of the sprint so it fits 3 months. Start these the week after — this is what turns "capable" into "strong hire".

CKA certification

The strongest infra credential — a hands-on exam fixing real clusters in a terminal. After the sprint you're ~3 weeks of practice away. Paid (~$400s; Linux Foundation runs 30–50% sales constantly). Prep free on Killercoda CKA scenarios.

AWS Solutions Architect – Associate

$150 exam, broad cloud credibility. Free prep: freeCodeCamp's ~50-hr course + AWS Skill Builder. CKA + SAA together reliably get infra interviews.

MLOps Zoomcamp (free + free certificate)

A free 9-week course that deepens your weeks 8–10: experiment tracking, orchestration, drift monitoring, deployment patterns.

Designing Data-Intensive Applications

The one book worth buying (~$40): how data survives across many servers — replication, partitioning, consensus. A chapter a week for a decade of payback.

Learn Go properly

Kubernetes, Docker, Prometheus and Terraform are written in Go — it's the native language of infra tooling and a real interview differentiator.

MIT 6.824 lectures (free)

The famous distributed-systems course, recorded. Watch MapReduce, Raft, and fault tolerance alongside the Kleppmann book.
